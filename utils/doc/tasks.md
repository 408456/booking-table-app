## Задачи в рамках проекта

Авторизовавшись, он проматривает список ресторанов с сортировкой
Выбирает доступные столики в ресторане(по id)
создает бронирование
Может оставить отзыв, отредактировать его, удалить
Может обновить свои данные


получение ресторанов по id кухни
получение всех бронирований по id ресторана связанных
получение всех бронирований пользователя
получение свободных столов ресторана 
получение всех столиков ресторана

### Описание нумерации задач

Задачи деляться следующие категории: 
- `API` - Разработка API-методов. Порядковый номер начинается с `1XXX`.
- `BD` - Миграции, доработка базы данных. Порядковый номер начинается с `2XXX`.
- `Test` - Написание интеграционных и юнит тестов. Порядковый номер начинается с `3XXX`.
- `Doc` - Написание документации. Порядковый номер начинается с `9XXX`.

Пример задачи:
Test-3001: Написать интерграционные тесты и юнит тесты для user controller, user service

### GitFlow
Создаются ветки для решения задач от ветки `develop`, после прохождения тестирования ветка мержиться в `develop`
После тестирования на dev окружении ветка мержиться в master.
Директории для создания веток:
- feature(для реализации нового функционала)
- fix(для доработки существующего в develop кода)
